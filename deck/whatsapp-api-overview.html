WhatsApp Business API
Direct Integrator Guide

This guide walks your team through everything needed to integrate WhatsApp messaging directly with Meta — from account provisioning to sending your first message in code. No middleman. Full control.

The WhatsApp Business Platform gives companies programmatic access to send and receive WhatsApp messages at scale via REST APIs. As a direct integrator, you integrate directly with Meta — no Business Solution Provider (BSP) needed — giving you full ownership of your WhatsApp Business Account, data, and costs.

☁️ Cloud API

Meta hosts the infrastructure. You call REST endpoints and receive events via webhooks. No servers to manage.

🏢 Direct Access

Own your WhatsApp Business Account directly with Meta. Skip BSP markup costs and retain full data control.

📬 At Scale

Send transactional alerts, OTPs, and notifications to millions of users — with delivery receipts and quality tracking.

Account Setup

Four things you need before your first API call

1

Create a Meta Business Account

Go to business.facebook.com and create or claim your company account using a company email. Then submit for Business Verification — upload your business registration or tax ID. Verification takes 1–5 business days and unlocks higher sending tiers.

Required Verification: 1–5 days
2

Create a WhatsApp Business Account + Phone Number

Inside Meta Business Suite, create a WhatsApp Business Account (WABA). Then add a dedicated phone number — it must not be active on any WhatsApp account. Verify via OTP. This is the sender number your customers will see.

Required Fresh number only
3

Create a Meta App + Generate a System Access Token

In the Meta Developers Portal, create a business-type app and add the WhatsApp product. Create a System User and generate a long-lived access token with the whatsapp_business_messaging permission. This token is your API key — store it securely.

Required developers.facebook.com
4

Set Up a Webhook Endpoint

Deploy a public HTTPS endpoint to receive delivery receipts, inbound messages, and account alerts. During setup Meta sends a one-time verification challenge your endpoint must echo back. Use ngrok for local testing.

Required Must be HTTPS
⚠️
Display Name Review: The sender name shown to customers must match your legal business name and goes through a Meta review. Submit it early — rejections require a full resubmission and add days to your timeline.

Tooling

Key platforms and APIs your team will work with

📊

Meta Business Suite Required

Admin dashboard for managing business accounts, phone numbers, display names, and business verification. Non-technical stakeholders (ops, compliance, finance) will use this most.

🛠️

Meta Developers Portal Required

Where engineers create apps, add the WhatsApp product, generate access tokens, and configure webhooks. All technical setup starts here.

📡

WhatsApp Cloud API Core API

The main REST API for sending messages, managing templates, and handling media. Base URL: https://graph.facebook.com/v19.0/. JSON in, JSON out.

📋

WhatsApp Manager Required

A section within Meta Business Suite for managing WABAs, phone numbers, message templates, and quality ratings. You'll use this regularly to submit and monitor templates.

🧪

Test Sandbox Optional

Meta provides a sandbox number in the Developer Portal. Use it to test the hello_world template before your real number is set up. Has restrictions on recipients.

💡
Use a System User token, not a personal token. Personal tokens expire and will break your integration. Generate a long-lived System User token in Meta Business Suite → System Users.

Local Environment Setup

Get your machine ready and send your first message in minutes

1. Gather your credentials

You need three values from the Meta Developer Portal. Find them under App → WhatsApp → API Setup.

PHONE_NUMBER_ID
The numeric ID of your registered sender number (not the phone number itself). Found in the API Setup panel next to your number.
WABA_ID
Your WhatsApp Business Account ID. Found in WhatsApp Manager or in the API Setup panel. Used for account-level operations like creating templates.
ACCESS_TOKEN
Your System User long-lived access token. Generate in Meta Business Suite → System Users with the whatsapp_business_messaging scope. Never commit this to source control.

2. Create your .env file

Store credentials as environment variables. Add .env to your .gitignore.

.env 
# .env — never commit this file
WHATSAPP_PHONE_NUMBER_ID=1234567890123456
WHATSAPP_WABA_ID=9876543210987654
WHATSAPP_ACCESS_TOKEN=EAAxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
WHATSAPP_API_VERSION=v19.0

3. Send your first message

Use Meta's built-in hello_world template to verify your setup end-to-end. Replace +1415XXXXXXX with your personal number (must be a WhatsApp-registered number).

bash 
curl -X POST \
  "https://graph.facebook.com/v19.0/$WHATSAPP_PHONE_NUMBER_ID/messages" \
  -H "Authorization: Bearer $WHATSAPP_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "messaging_product": "whatsapp",
    "to": "+1415XXXXXXX",
    "type": "template",
    "template": {
      "name": "hello_world",
      "language": { "code": "en_US" }
    }
  }'
javascript 
// npm install dotenv  (fetch built-in in Node 18+)
import dotenv from 'dotenv';
dotenv.config();

const { WHATSAPP_PHONE_NUMBER_ID, WHATSAPP_ACCESS_TOKEN, WHATSAPP_API_VERSION } = process.env;

const res = await fetch(
  `https://graph.facebook.com/${WHATSAPP_API_VERSION}/${WHATSAPP_PHONE_NUMBER_ID}/messages`,
  {
    method:  'POST',
    headers: {
      'Authorization': `Bearer ${WHATSAPP_ACCESS_TOKEN}`,
      'Content-Type':  'application/json',
    },
    body: JSON.stringify({
      messaging_product: 'whatsapp',
      to:       '+1415XXXXXXX',
      type:     'template',
      template: { name: 'hello_world', language: { code: 'en_US' } },
    }),
  }
);

console.log(await res.json());
// → { messages: [{ id: 'wamid.HBgL...' }] }
python 
# pip install requests python-dotenv
import os, requests
from dotenv import load_dotenv

load_dotenv()

resp = requests.post(
    f"https://graph.facebook.com/{os.environ['WHATSAPP_API_VERSION']}"
    f"/{os.environ['WHATSAPP_PHONE_NUMBER_ID']}/messages",
    json={
        "messaging_product": "whatsapp",
        "to":       "+1415XXXXXXX",
        "type":     "template",
        "template": {"name": "hello_world", "language": {"code": "en_US"}},
    },
    headers={"Authorization": f"Bearer {os.environ['WHATSAPP_ACCESS_TOKEN']}"},
)
resp.raise_for_status()
print(resp.json())  # {"messages": [{"id": "wamid.xxx"}]}
✅
Success: A 200 OK with {"messages":[{"id":"wamid.xxx"}]} means Meta accepted the message. Check your phone — it should arrive within seconds.

5. Test webhooks locally with ngrok

Meta needs a public HTTPS URL to push events to. During local development, use ngrok to tunnel your local port.

bash 
# Install ngrok: https://ngrok.com/download
# Start your local server first (e.g. port 3000), then:
ngrok http 3000

# ngrok outputs a public URL, e.g.:
#   https://abc123.ngrok-free.app
#
# In Meta Developer Portal → App → WhatsApp → Configuration:
#   Webhook URL:   https://abc123.ngrok-free.app/webhook
#   Verify Token:  any string you choose (must match your handler)
javascript 
// npm install express
import express from 'express';
const app = express();
app.use(express.json());

const VERIFY_TOKEN = 'my-secret-verify-token'; // match in Meta portal

// Meta calls GET to verify the endpoint
app.get('/webhook', (req, res) => {
  const { 'hub.mode': mode, 'hub.verify_token': token, 'hub.challenge': challenge } = req.query;
  if (mode === 'subscribe' && token === VERIFY_TOKEN) {
    res.send(challenge); // echo back the challenge to verify
  } else {
    res.sendStatus(403);
  }
});

// Meta sends POST for all events
app.post('/webhook', (req, res) => {
  res.sendStatus(200); // always ack first (within 20s)
  const value = req.body?.entry?.[0]?.changes?.[0]?.value;
  if (value?.messages) {
    const msg = value.messages[0];
    console.log('Inbound:', msg.from, msg.text?.body);
  }
  if (value?.statuses) {
    const s = value.statuses[0];
    console.log('Status:', s.id, s.status); // sent | delivered | read
  }
});

app.listen(3000);
💡
Sandbox shortcut: The Meta Developer Portal has a built-in "Send message" button under WhatsApp → API Setup that triggers the hello_world template using Meta's test number — useful to confirm your account is active before writing any code.

Message Templates

Required for all business-initiated conversations outside the 24h service window

When you initiate a conversation (not replying to a user message), you must use a pre-approved template. Templates are submitted via WhatsApp Manager and reviewed by Meta.

Category Examples Approval
Authentication OTPs, login codes Minutes (automated)
Utility Order confirmations, ride status, shipping updates Minutes–hours
Marketing Promotions, offers, re-engagement Hours–2 days

Template structure

  • Header — Optional text, image, video, or document
  • Body — Main text with variables: {{1}}, {{2}}
  • Footer — Optional small text
  • Buttons — CTA or Quick Reply (optional)

Common rejection reasons

  • Promotional content in a Utility template
  • Vague or misleading variable placeholders
  • Missing opt-out language on Marketing templates
  • Unclear business purpose or off-brand naming

Webhooks & Inbound Handling

How Meta delivers real-time events to your system

Meta sends all events as HTTP POST requests to your configured webhook URL. Always respond with 200 OK immediately, then process asynchronously.

Event Fires when What to do
messages User sends you a message Route to support system or autoresponder
statuses Message sent / delivered / read / failed Update delivery state in your database
errors Delivery failure Log, alert, retry or mark as failed
account_alerts Quality rating or limit changes Notify your ops team
⚠️
Idempotency: Meta may deliver the same event more than once. Always deduplicate by message ID before processing.

Messaging Limits & Quality Rating

How Meta controls how many users you can reach per day

Limits apply to unique users messaged per rolling 24 hours. You start at Tier 1 and scale up automatically based on volume and quality.

Quality Rating

Based on user reports and blocks. High = grow freely. Medium = monitor. Low = risk of restriction.

High Medium Low

Protecting Your Rating

  • → Only message opted-in users
  • → Include clear opt-out instructions
  • → Respect opt-outs immediately
  • → Keep messages timely and relevant

Pricing Model

Billed per conversation window, not per message

A conversation is a 24-hour messaging window. Once open, you can send unlimited messages within it at no extra cost.

Type Initiated by Cost Notes
Marketing Business Higher Promos, re-engagement
Utility Business Medium Transactional notifications
Authentication Business Medium-low OTPs, verification
Service User Free* Reply within 24h user-initiated window

* First 1,000 service conversations per month are free. Pricing varies by country — check Meta's pricing page for your markets.

💡
Cost tip: When a user messages you first, a free 24-hour service window opens. Use it to resolve issues or send follow-ups without paying for a business-initiated conversation.

Pre-Launch Checklist

Confirm all of these before going live

Account & Identity

  • Meta Business Account created
  • Business Verification approved
  • WhatsApp Business Account (WABA) created
  • Display Name reviewed and approved
  • Dedicated phone number registered

Technical

  • Meta App created with WhatsApp product
  • System User long-lived token generated
  • Webhook endpoint deployed and verified
  • Webhook deduplication implemented
  • Message templates approved

Compliance

  • Opt-in mechanism in place for all recipients
  • Opt-out flow implemented and tested
  • Privacy policy updated
  • Message log retention policy defined

Operations

  • Quality rating monitoring configured
  • Runbook for failed / blocked messages
  • On-call process if account gets limited
  • Load test within allowed tier limits
